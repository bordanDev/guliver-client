# Rules

- ChangeDetection: Всегда `ChangeDetectionStrategy.OnPush`.
- Реактивность: Не использовать `setInterval`, получать данные через `Observable`, в UI использовать `async` pipe
- Standalone: Все компоненты должны быть `standalone: true`.
- Подписки: Использовать `takeUntilDestroyed` или `DestroyRef` для защиты от утечек памяти
- Версионирование: Angular и PrimeNG версии должны точно совпадать (например, `20.x`).
- Контрол Флоу: Использовать новый подход с @if, @for, @switch
- Импорты и пути: Создавать для каждой сущности/вложенности/папки index.ts для удобного импорта
- Стилевую и классовую логику в виде ngClass/ngStyle писать в scss файлах или на уровне ts файла в виде геттеров/методов
- При создании компонент/сервисов/модулей и прочих сущностей, ты должен записывать это в GRAPH.md и добавлять/обновлять статус в файле COMPONENTS.md для сохранения контекста проекта
- Каждая фича должна иметь свой module.ts файл для поставки
